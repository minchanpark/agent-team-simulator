# PRD — AI 에이전트 팀 시뮬레이터

## 1. 프로젝트 개요

### 서비스 한 줄 설명
아이디어만 있는 예비창업자가 자신의 스타트업에 맞는 AI 에이전트 팀을 구성하고, 직접 대화해보고, 실제 구현 청사진까지 받아가는 웹 서비스

### 목적
- 포트폴리오 완성도 우선
- Claude API + Next.js 기반 AI 서비스 구현 능력 시연
- 창업/AI 에이전트 도메인 전문성 표현

---

## 2. 타겟 사용자

- **주요 타겟:** 아이디어는 있지만 팀이 없는 예비창업자 (비개발자)
- **페인 포인트:** "AI로 뭘 자동화할 수 있는지 알고 싶지만, 어디서부터 시작해야 할지 모르겠다"
- **기대 가치:** 실제로 AI 팀원과 대화해보는 경험 → 막연함 해소 → 실행 동기 부여

---

## 3. 핵심 기능 (MVP 범위)

### Feature 1 — 스타트업 진단 (온보딩)
- 3스텝 질문 폼
  - Step 1: 아이디어 한 줄 설명 (텍스트 입력)
  - Step 2: 힘든 업무 선택 (멀티 체크박스)
  - Step 3: 팀 규모/단계 선택 (라디오)
- 입력값은 이후 에이전트 컨텍스트에 주입됨

### Feature 2 — 에이전트 팀 구성 추천
- 진단 결과 기반으로 3~4개 에이전트 카드 자동 생성
- 각 카드 포함 정보:
  - 에이전트 이름 + 이모지 아이콘
  - 역할 한 줄 설명
  - 할 수 있는 업무 3가지
  - [대화하기] 버튼

### Feature 3 — 에이전트와 대화 (채팅 인터페이스)
- 카드 클릭 시 사이드패널 or 모달로 채팅 UI 오픈
- 선택한 에이전트 페르소나로 Claude API 응답
- 대화 히스토리 유지 (세션 내)
- 채팅 하단에 [구현 청사진 보기] 버튼 고정

### Feature 4 — 구현 청사진
- 에이전트별 실제 구현 방법 카드 표시
  - 필요한 도구 (Claude API, n8n, Zapier 등)
  - 구현 단계 (3~5단계)
  - 예상 비용 / 난이도

## 4. 페이지 구조

```
/                → 랜딩 페이지 (히어로 + CTA)
/onboarding      → 진단 3스텝 폼
/result          → 에이전트 팀 카드 목록
/result/[agent]  → 개별 에이전트 채팅 (동적 라우트 or 모달)
/blueprint/[agent] → 구현 청사진
```

| 항목 | 기술 |
|------|------|
| 프레임워크 | Next.js 14 (App Router) |

## 6. 에이전트 종류 (MVP)

|----------|--------|-----------|
| 마케팅 에이전트 | 🧲 | SNS 콘텐츠, 카피라이팅 |

| 개발보조 에이전트 | 🛠 | 노코드 추천, PRD 작성 |

## 7. 에이전트 추천 로직

진단 결과의 `pain_points` 값에 따라 에이전트 우선순위 결정:

```
콘텐츠/마케팅 선택 → 🧲 마케팅 에이전트 (1순위)
고객 응대 선택     → 💬 CS 에이전트 (1순위)
데이터 분석 선택   → 📊 데이터 에이전트 (1순위)
기술 개발 선택     → 🛠 개발보조 에이전트 (1순위)
```

나머지 에이전트는 2~3순위로 함께 표시.

## 8. API 설계

### POST /api/chat
```json
Request:
{
  "agentType": "marketing" | "cs" | "data" | "dev",
  "messages": [{ "role": "user" | "assistant", "content": "string" }],
  "context": {
    "idea": "string",
    "painPoints": ["string"],
    "teamSize": "solo" | "small" | "early"
  }
}

Response:
{
  "message": "string"
}
```

## 9. 디자인 방향

- **톤:** 전문적이지만 친근함. 딱딱한 SaaS보다 따뜻한 스타트업 느낌
- **컬러:** 다크 베이스 or 화이트 베이스 + 포인트 컬러 1개 (보라/인디고 계열 추천)
- **레이아웃:** 모바일 대응 (반응형 필수)
- **애니메이션:** 카드 등장, 채팅 스트리밍 효과 (타이핑 느낌)
